# Como Contribuir

Obrigado por considerar contribuir para este projeto!  
Siga as orientações abaixo para garantir que o processo seja simples e organizado.

---

## ✅ Reportando Problemas (Issues)

Se você encontrou um bug, comportamento inesperado ou tem uma sugestão:

1. Abra uma **Issue**.
2. No titulo use palavras chaves (modulo , erro , etc) e dê uma breve descrição. Ex:
    - [Product] arrumando o erro na permissão de edição dos produtos
    - [settings] arrumando as configurações do cache
3. Descreva claramente o problema.
4. Inclua passos para reproduzir, se possível.
5. Adicione prints, logs ou exemplos de código quando necessário.

---

## ✅ Fazendo um Pull Request

Se deseja enviar uma melhoria ou correção:

1. **Faça um fork** do repositório.
2. Crie uma nova branch:
```bash
git checkout -b minha-melhoria
```
3. Faça suas alterações.
4. Garanta que tudo está funcionando.
5. Tente aplicar testes automatizados ou rode os existentes para ter certeza que a aplicação continua funcionando como deveria
6. Envie o PR com uma descrição clara do que foi alterado e se possivel fale a issue (caso for correção).
---

## ✅ Padrões de Código

- Mantenha o estilo de código consistente com o projeto.
- Nomeie variáveis e funções de forma clara.
- Evite adicionar dependências desnecessárias.
- Utilize ou crie mixins para a reutilização de logica se necessario
- Se possivel utilize docstrings para documentar o que a class/função/modulo faz

---

## ✅ Estrutura de Commits

Use mensagens de commit claras e objetivas. Exemplos:

- `fix: corrige erro no login`
- `feat: adiciona validação de email`
- `docs: atualiza README`
- `fetuare : novo componente/modulo "simples descrição"`
---

## ✅ Dúvidas

Se tiver qualquer dúvida, abra uma Issue com a tag **question**.

---

## ✅ Melhorias

Se tiver qualquer sugestão de uma possivel melhoria , abra uma Issue com a tag **ideia**.

---

Agradeço sua contribuição!
